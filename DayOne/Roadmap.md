# 🚀 Java & OOP Mastery Roadmap

## Phase 1: The "Grammar" (Basic Syntax)
*Goal: Learn how to write valid Java statements.*
- [ ] **Program Structure:** `public static void main(String[] args)`
- [ ] **Variables & Types:** Primitives (`int`, `double`, `boolean`) vs. `String`.
- [ ] **Operators:** Arithmetic (`+`, `-`, `*`, `/`, `%`) and Logical (`&&`, `||`, `!`).
- [ ] **Input/Output:** Using `System.out.println()` and the `Scanner` class for user input.
- [ ] **Control Flow:** `if-else`, `switch` blocks, and loops (`for`, `while`, `do-while`).

## Phase 2: Methods & Memory
*Goal: Understand how data moves through a program.*
- [ ] **Method Anatomy:** Return types, parameters, and the `return` keyword.
- [ ] **Method Overloading:** Defining multiple methods with the same name but different parameters.
- [ ] **Scope:** Local variables vs. Instance variables.
- [ ] **The Stack vs. The Heap:** How Java manages memory for primitives vs. objects.

## Phase 3: The Four Pillars of OOP (The "Deep Details")
*Goal: Master the core logic of Object-Oriented Programming.*

### 1. Encapsulation (Data Hiding)
- [ ] **Access Modifiers:** `private`, `public`, `protected`, and default.
- [ ] **Getters & Setters:** Protecting internal data.
- [ ] **Constructors:** Using `this()` and initializing objects.

### 2. Inheritance (Hierarchy)
- [ ] **The `extends` Keyword:** Creating child classes.
- [ ] **Method Overriding:** Using `@Override` to change parent behavior.
- [ ] **The `super` Keyword:** Calling parent constructors and methods.

### 3. Polymorphism (Flexibility)
- [ ] **Static vs. Dynamic Binding:** Understanding how Java decides which method to run.
- [ ] **Upcasting & Downcasting:** Treating a `Dog` as an `Animal`.

### 4. Abstraction (Design)
- [ ] **Abstract Classes:** Methods without bodies.
- [ ] **Interfaces:** Using `implements` to define a "contract" for behavior.

## Phase 4: Advanced Syntax & Error Handling
*Goal: Write "bulletproof" Java code.*
- [ ] **Exception Handling:** `try`, `catch`, `finally`, and `throw`.
- [ ] **Packages & Imports:** Organizing large projects into folders.
- [ ] **Static vs. Instance:** Understanding the `static` keyword (belongs to the class, not the object).

## 🛠️ Weekly Study Habit
1. **Lecture:** Take notes on the *concept* (The "Why").
2. **VS Code:** Write 10-20 lines of code testing that concept (The "How").
3. **Debug:** Use the VS Code debugger to watch how variables change line-by-line.
